# self-evolution

`skyfix` 的自我进化在 V1 只做到：**复盘 + 提案**。

## V1 允许的事
- 维护后回顾这次修复流程
- 提炼哪些步骤稳定、哪些步骤脆弱
- 生成后续改进建议
- 为未来脚本化、文档化提供提案

## V1 不允许的事
- 自动修改 `skyfix` 自己
- 自动发布新版本
- 自动替换主流程

## 与 `self-improving-agent` 的关系
- `self-improving-agent` 是 `skyfix` 的进化外挂
- 用于维护结束后的模式抽象与改进建议
- `skyfix` 本体仍然保留主编排权

## 与 `auto-skill` 的边界
- `auto-skill` 可以提供经验记录思路
- 但不进入 `skyfix` 主运行链
- 不允许 `auto-skill` 的全局强制协议覆盖 `skyfix`

## 输出标准
每次复盘至少回答：
- 哪些步骤重复出现？
- 哪些动作值得脚本化？
- 哪些判断最容易出错？
- 下次应该怎样更快修到位？
