# process-discipline

来源：内化 `superpowers-openclaw` 的方法论，但由 `skyfix` 自己执行，不依赖其运行期存在。

## 核心纪律
1. 先判断，再动手
2. 先计划修法，再改文件/服务
3. 先找根因，再给修复
4. 先验证，再说完成

## 对 skyfix 的具体要求

### 1. 不瞎重装
除非有理由，否则先判断是否能轻修：
- 配置修
- service 重建
- 模型/权限收口
- 平台链路修复

### 2. 根因优先
遇到 bug：
- 先看日志
- 先判断问题层级
- 不要一上来就给猜测性补丁

### 3. 证据优先
在说“修好了”前，至少给一个当前证据：
- 版本输出
- validate 通过
- 端口监听
- 平台发送成功
- 语音链路成功

### 4. 过程压缩但不虚假
可以简洁，但不能跳过关键判断。

## 适用场景
所有 `skyfix` 任务都默认遵循这份纪律。
