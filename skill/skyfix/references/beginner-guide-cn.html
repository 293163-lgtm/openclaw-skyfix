Quick Start

30 秒上手

1
直接说人话，不用背命令。
帮我把 OpenClaw 装好 把 em2t 那台机器的 OpenClaw 收口 修一下 Telegram 发不出去的问题
2
它会先判断问题层级，再动手。

不是一上来就重装。它会先区分：配置问题、权限问题、gateway 问题、网络问题、平台问题、依赖缺失。

3
修完必须给结果，不靠嘴硬。

会说明现在版本、主模型、权限状态、gateway / LaunchAgent 状态、平台接入状态、剩余问题。

最简单的理解：你把需求说清楚，skyfix 负责把 OpenClaw 的维护链路接住。
Scope

它到底能做什么

安装 / 初始化

新机部署 OpenClaw、检查 Node / PATH、放好工作目录、补桌面快捷方式。

升级 / 重装 / 修复

版本升级、坏状态修复、gateway / LaunchAgent 拉起、配置混乱时做收口。

权限 / 模型 / 配置收口

切主模型、补 API key、统一 exec 权限与 elevatedDefault，确保配置能过校验。

Telegram / 语音 / 文件发送

区分 token、网络、发送策略、语音依赖缺失，不把所有锅都甩给 Telegram。

微信在 V1 只做 lite 诊断，不承诺全自动修好。这个边界是故意留的，不是偷懒。
Examples

直接照着说就行

装机类

帮我在这台机器安装 OpenClaw 给我把 OpenClaw 初始化好，顺手放桌面快捷方式

修复类

OpenClaw 起不来，帮我修 gateway 挂了，帮我把 LaunchAgent 收口

配置类

把主模型切到 nowcoding/gpt-5.4 把权限开到最高，并验证配置没坏

平台类

看看 Telegram 为什么发不出去 帮我查查语音回复链路缺了什么
如果是远端机器，也直接说：去 em2t 上做、把这台机器收口。它会走远端维护链路。
How It Works

它会怎么做事

A
先检查，不先瞎修。

先看当前版本、配置、主模型、权限、gateway / LaunchAgent、平台接入状态。

B
再判断该轻修、重装还是只收口。

有些问题只需要补配置；有些问题要修服务；有些问题其实是网络或依赖没装，不该上来就删库重来。

C
最后做验收。

修完要验证，没验证就不算完成。它会明确告诉你现在通了没，哪里还卡着。

它不会把“看上去像好了”当作“真的修好了”。这点很重要。
Output

你最后会拿到什么结果

  • 当前 OpenClaw 版本
  • 当前主模型
  • 当前权限状态
  • gateway / LaunchAgent 状态
  • 如果涉及 Telegram / 语音：会说明卡在哪一层
  • 关键日志路径 / 配置路径
  • 剩余问题和下一步建议
一句人话版： 不是只告诉你“我试过了”，而是告诉你“我修了什么、现在怎样、还差什么”。
Boundaries

哪些事它不会乱做

不会一上来粗暴重装

先诊断、先备份、再决定是轻修还是重装。

不会把别的 skill 主控协议带进来

skyfix 是唯一维护编排器。它可以吸收别人的能力，但不会让外部协议反客为主。

不会把微信吹成全自动修复

V1 只承诺 lite 诊断，这个边界是清楚写死的。

不会没验证就宣布胜利

没有验证就没有完成，这条铁律不打折。

skyfix / 使用建议
最好把它当成“OpenClaw 维护专家”，不是通用电脑维修工。你让它装、修、收口、验收，它会很专业；你让它处理完全无关的杂务，那就偏题了。
